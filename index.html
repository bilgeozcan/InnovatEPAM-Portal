    Portal Overview

    InnovatEPAM Portal is an employee innovation management platform that lets employees submit ideas, track progress, and get structured feedback from evaluators — all with a visible audit trail.

    01

    Submit an idea

    Any registered submitter can create an idea with a title, description, category, and optional file (max 2 MB).

    02

    Track progress

    Watch your idea move through: Submitted → Under Review → Accepted or Rejected.

    03

    Get feedback

    Every status change includes an evaluator comment explaining the decision clearly.

    Quick start

    1. Sign in or register — use demo accounts or create your own account
    2. Choose your role — Submitter to create, Evaluator Admin to review
    3. Submit your first idea — go to Submit Idea and fill the form
    4. Track on dashboard — see status and full decision history
    Demo accounts: Use admin@innovatepam.local / Admin123! or aylin@epam.local / Submit123! to explore both roles instantly.

    Submitter Guide

    As a submitter, you create and track your own ideas. Here's everything you need to know to make the most of the portal.

    Creating a strong idea

    1. Go to Submit Idea in the top navigation
    2. Enter a clear, actionable title (5–90 characters)
    3. Select the most relevant category from the dropdown
    4. Write a description that covers: the problem, your proposed solution, and expected impact (min 20 chars)
    5. Optionally attach a support file — any format, max 2 MB
    6. Click Submit idea — it enters the review pipeline immediately
    Pro tip: Ideas with clear impact statements and an attached file are significantly more likely to be accepted. Evaluators respond better to evidence than to assertions.

    Available categories

    • AI and Automation — AI/ML tools, automation scripts, intelligent workflows
    • Customer Experience — improvements to client-facing products or services
    • Developer Productivity — dev tooling, CI/CD improvements, code quality
    • Process Improvement — workflow optimizations, cross-team efficiency
    • Sustainability — environmental impact, energy efficiency, ESG initiatives
    • Workplace Experience — office environment, remote work, culture

    Tracking your ideas

    Visit the Dashboard to see all your submitted ideas. Use the search bar and status filter to narrow results. Expand the Status history section on each card to see the full audit trail.

    Note: Submitters see only their own ideas. Evaluator Admins see all ideas from all users.

    Evaluator Admin Guide

    As an evaluator admin, you review and make decisions on all submitted ideas. Your feedback shapes the innovation culture of the organization.

    Reviewing ideas effectively

    1. Go to Dashboard — you see all ideas across all submitters
    2. Use the status filter to find ideas in "Submitted" status
    3. Read the idea description and download any attached files
    4. Use the review form at the bottom of each idea card
    5. Select a new status — Under Review, Accepted, or Rejected
    6. Add a clear evaluator comment — required for Accept/Reject decisions
    7. Click Update — submitter sees the change immediately
    Important: A comment is required when accepting or rejecting. Write a genuine explanation — submitters use this feedback to improve future ideas.

    Review workflow

    Submitted → Under Review → Accepted
    Under Review → Rejected

    Best practices

    • Review all attachments before making a decision
    • Use "Under Review" to signal that an idea has been noticed before making a final call
    • Always explain why — not just "rejected" but what would make the idea stronger
    • Aim to review all "Submitted" ideas within a reasonable timeframe

    Status Workflow

    Every idea moves through a defined set of statuses, each with a specific meaning and set of rules.

    Submitted

    Submitted

    Idea created and in the queue, waiting for an evaluator to begin review.

    Set by: System (automatic on create)

    Under Review

    Under Review

    Rejected

    Frequently Asked Questions

    Can I edit an idea after submitting?

    Not in the current MVP version. Plan your submission carefully before clicking Submit. Future versions will include draft management (save and edit before submitting).

    What file types can I attach?

    Any file type is supported, with a maximum size of 2 MB. Useful formats include PDF, DOCX, XLSX, PNG, JPG, or TXT.

    Can I submit multiple ideas?

    Yes! There's no limit on the number of ideas you can submit. Each idea is tracked independently through the workflow.

    Why can't I see other users' ideas?

    Submitters can only see their own ideas. Evaluator Admins see all ideas across all users — this is by design so submitters can't copy each other's ideas before review.

    Is my data saved between sessions?

    Yes, data is stored in your browser's localStorage. Clearing browser data or clicking "Reset demo" will erase everything. This is a browser-local MVP.

    How do I switch between roles?

    Log out and log in with a different account. Use the demo buttons on the sign-in page for quick role switching between Admin and Submitter.

    I see my password in the URL — is that a bug?

    This would be a browser autofill artifact. All forms now use method="post" which prevents form data from appearing in URLs. Passwords are never stored in plain text — only a hashed version is kept.
